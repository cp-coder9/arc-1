# Architex Navigation Framework

## A. Top-level sidebar

The sidebar should contain only primary platform zones:

1. Command Centre
2. Inbox
3. Projects
4. Toolboxes
5. CPD & Learning
6. Documents
7. Marketplace
8. Finance
9. Messages
10. Settings

## B. Workspace structure

### 1. Command Centre

Purpose: personalised daily landing page.

Sections:
- Today / next actions
- Active projects
- CPD status
- Pending approvals
- Payment alerts
- Unread priority messages
- Agent recommendations
- Recent project activity

### 2. Inbox

Purpose: all user action cards from agents and workflows.

Sections:
- Required actions
- Approvals
- Retakes / resubmissions
- Project tasks
- CPD tasks
- Finance tasks
- Documents requiring attention
- Overdue items

### 3. Projects

Purpose: project-specific workspace.

Sections:
- Project dashboard
- Phases
- Team
- Programme
- Documents
- Project messages
- Appointments
- Deliverables
- RFIs
- Instructions
- Variations
- Site diary
- Snags
- Payments
- Closeout
- Audit trail

Phase-aware menu:
- Brief / onboarding
- Feasibility
- Proposal / appointment
- Design
- Council / statutory
- Tender
- Procurement
- Construction
- Payment / drawdowns
- Snagging
- Closeout
- Archive

### 4. Toolboxes

Purpose: professional tools that may be used standalone or inside a project.

Sections:
- Proposal & appointment tools
- Design & compliance tools
- Costing / procurement tools
- Construction admin tools
- Closeout tools
- Practice / business tools
- Full tool library

### 5. CPD & Learning

Purpose: professional learning, assessments, CPD records and accreditation workflows.

Sections:
- CPD dashboard
- My CPD cycle
- Courses / webinars
- Articles / reading CPD
- Assessments
- Certificates
- Evidence packs
- Manual submissions
- CPD Central / partners
- Lecturer/content creator tools
- Reviewer tools
- Accreditation coordinator tools
- Partner admin

### 6. Documents

Purpose: global document and knowledge hub.

Sections:
- My documents
- Project documents
- Drawing packs
- Submission packs
- Templates
- Contracts / appointments
- Reports
- Certificates
- Knowledge articles
- Compliance references
- Version history

### 7. Marketplace

Sections:
- Profile
- Professional registrations
- Company settings
- Team settings
- Roles / permissions
- Billing / subscription
- Notification preferences
- Agent preferences
- Admin configuration
- CPD body rules
- Tool configuration

## C. Module placement rules

1. CPD is its own platform zone, not a toolbox.
2. Snags, RFIs, instructions, variations and site records primarily live inside Projects.
3. Toolboxes provide tools, but project-linked outputs should be written back into the relevant project.
4. Finance has a global view, but project financial items also appear inside the project.
5. Documents has a global view, but project documents also appear inside the project.
6. Messages has a full centre, but quick messaging is exposed inside workflows.
7. Admin/settings must be role-gated.
8. Marketplace is separate from active project management.
9. Inbox is where action items converge.
10. Command Centre is the user’s daily cockpit.
